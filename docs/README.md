# Глобальная структура проекта
Корневой объект описывает общую мета-информацию о решаемой проблеме и содержит списки ресурсов и задач
| Поле | Тип | Обязательно | Описание | 
|------|-----|-------------|----------|
|`schema_version`|`String`|Да|Версия схемы DSL (текущая: "0.1"). Позволяет управлять обратной совместимостью форматов при развитии языка|
|`problem_id`|`String`|Да|Уникальный идентификатор задачи (например `"T01_CLASSIC_SCALED"`)|
|`domain`|`String`|Да|Указание предметной области для парсера. Допустимые значения: `"rcpsp"`, `"cluster"`|
|`description`|`String`|Нет|Описание сценария|
|`project`|`Object`|Да|Объект с глобальными целями (см. структуру ниже)|
|`extensions`|`Object`|Нет|Блок для глобальных параметров расширения (например, `locations`, `objectives`)|
|`resources`|`Array`|Да|Массив доступных ресурсов|
|`tasks`|`Array`|Да|Массив работ|

## Структура `project`
| Поле | Тип | Описание | 
|------|-----|----------|
|`name`|`String`|Название проекта|
|`objective`|`String`|Главная цель оптимизации (например, `"minimize_makespan"`, `"minimize_completion_time"`). Если целей несколько, используется значение `"multi_objective"`|

# Ресурсы
Описывают всё, что накладывает ограничения на выполнение проекта: людей, бюджет, физическое пространство или вычислительные узлы
## Ядро ресурса
| Поле | Тип | Обязательно | Описание | 
|------|-----|-------------|----------|
|`id`|`String`|Да|Уникальный идентификатор ресурса (например, `"R_WORKER"` или `"NODE_TYPE_A"`)|
|`name`|`String`|Нет|Название ресурса|
|`capacity`|`Number`|Да|Базовая вместимость/доступное количество ресурса в системе|
|`extensions`|`Object`|Нет|Блок для специфичных параметров выбранного домена|

## Расширение `extensions.rcpsp`
| Поле | Тип | Описание | 
|------|-----|----------|
|`type`|`String`|Тип расходования: `"renewable"` (люди, техника) или `"non_renewable"` (материалы, бюджет)|
|`cost_per_period`|`Number`|Стоимость использования ресурса за единицу времени|
|`cost_per_unit`|`Number`|Стоимость использования одной единицы ресурса|
|`is_zone`|`Boolean`|Флаг, указывающий, что ресурс является физическим пространством|
|`is_global_capacity`|`Boolean`|Ограничивает общее количество юнитов на площадке одновременно|
|`location_dependent`|`Boolean`|Указывает, что ресурс привязан к локации и требует времени на перемещение|
|`initial_location`|`String`|Стартовая площадка для ресурса|
|`transfer_times`|`Object`|Матрица задержек на перемещение: `{"Откуда": {"Куда": Время}}`|

## Расширение `extensions.cluster`
В кластерном домене ресурсы обычно представляют собой серверные узлы (Nodes), и их capacity указывает на количество таких узлов в пуле
| Поле | Тип | Описание | 
|------|-----|----------|
|`gpu_type`|`String`|Модель графического ускорителя (например, `"A100"`, `"V100"`)|
|`gpu_memory_gb`|`Number`|Объем видеопамяти на одном узле (в гигабайтах)|
|`ram_gb`|`Number`|Объем оперативной памяти на одном узле (в гигабайтах)|
|`cpu_cores`|`Number`|Количество процессорных ядер на одном узле|

# Задачи
Описывают узлы графа работ. Специфика выполнения задачи, её длительность и требования полностью зависят от выбранного домена в `extensions`
## Ядро задачи
| Поле | Тип | Обязательно | Описание | 
|------|-----|-------------|----------|
|`id`|`String`|Да|Уникальный идентификатор задачи|
|`name`|`String`|Нет|Название задачи|
|`dependencies`|`Array`|Да|Массив предшествующих задач|
|`extensions`|`Object`|Да|Блок домена (обязателен для хранения длительности и требований)|

## Расширение `extensions.rcpsp`
| Поле | Тип | Описание | 
|------|-----|----------|
|`location`|`String`|Место выполнения работы|
|`modes`|`Array`|Список альтернативных режимов выполнения задачи. Включает `mode_id`, `duration` и массив `requirements` (`resource_id` + `quantity`)|

## Расширение `extensions.cluster`
| Поле | Тип | Описание | 
|------|-----|----------|
|`duration`|`Number`|Оценочное время выполнения вычислительной задачи|
|`is_preemptible`|`Boolean`|Флаг, разрешающий прерывание задачи планировщиком для освобождения ресурсов под более приоритетные джобы|
|`requirements`|`Array`|Массив требований к железу для данной задачи (см. структуру ниже)|

## Структура `requirements` внутри `extensions.cluster`
Каждый объект в массиве `requirements` определяет, сколько мощностей нужно выделить из конкретного пула ресурсов (`resource_id`)
- `resource_id` (`String`): ID требуемого узла
- `gpu_count` (`Number`): Требуемое количество GPU
- `cpu_cores` (`Number`): Требуемое количество ядер CPU
- `ram_gb` (`Number`): Требуемый объем оперативной памяти

# Зависимости
Описывают связи между задачами и топологическую сортировку. Располагается внутри `task.dependencies`
| Поле | Тип | Обязательно | Описание | 
|------|-----|-------------|----------|
|`task_id`|`String`|Да|ID задачи-предшественника|
|`type`|`String`|Да|Тип связи. Поддерживается `"FS"` (Finish-to-Start) и `"SS"` (Start-to-Start)|
|`extensions`|`Object`|Нет|Дополнительные параметры связи. В RCPSP может включать поле `lag` (задержка после завершения предшественника)|
